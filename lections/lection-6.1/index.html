    Базы данных. Основы SQL

    Базы данных и SQL

    База данных (БД) - это организованная коллекция данных, которая обычно хранится и обрабатывается с помощью компьютерных систем. Она представляет собой структурированный набор информации, организованный таким образом, чтобы обеспечить эффективный доступ, управление и обновление данных.

    Базы данных могут быть различными по своей природе, структуре и использованию. Они могут быть реляционными, иерархическими, сетевыми, объектно-ориентированными и т.д., в зависимости от способа организации данных внутри них и способа доступа к этим данным.

    Реляционная база данных

    Реляционная база данных - это тип базы данных, который организован на основе концепции реляционной модели данных. В реляционной модели данные представляются в виде таблиц, состоящих из строк и столбцов.

    Основные характеристики реляционных баз данных:

    • Таблицы: Данные организованы в виде таблиц, где каждая строка представляет собой запись, а каждый столбец - атрибут или поле. Таблицы могут быть связаны между собой с помощью ключевых столбцов.
    • Ключи: Ключи используются для идентификации строк в таблицах. Ключевой столбец в таблице может быть определен как первичный ключ, который гарантирует уникальность каждой строки в таблице.
    • Отношения: Отношения между таблицами устанавливаются с помощью внешних ключей. Это позволяет связывать данные из разных таблиц и создавать сложные запросы для извлечения информации из базы данных.
    • Язык запросов: Для работы с данными в реляционных базах данных используется стандартизированный язык запросов, который предоставляет способы выполнения запросов к данным, вставки, обновления и удаления записей.

    SQL (Structured Query Language)

    SQL (Structured Query Language) - это специализированный язык программирования, который используется для управления данными в реляционных базах данных. Он предоставляет стандартизированный способ выполнения операций с данными, таких как запросы, вставки, обновления и удаления.

    SQL был разработан в IBM в конце 1970-х годов в ответ на необходимость стандартизированного языка запросов для управления данными в реляционных базах данных. С тех пор он стал стандартом ISO и ANSI, последняя версия стандарта - SQL:2023, принятая в июне 2023 года.

    СУБД (система управления базами данных)

    СУБД (система управления базами данных) - это программное обеспечение, которое используется для создания и управления базами данных. СУБД предоставляют средства для хранения, организации, обновления и извлечения данных из базы данных.

    MySQL

    MySQL Server — это промышленная система управления базами данных (СУБД), сопровождаемая компанией Oracle. Она позволяет пользователям создавать, модифицировать и управлять базами данных, а также обеспечивает безопасность и целостность данных.

    Сервер MySQL — центральный компонент системы, который выполняет основную работу по обработке запросов, управлению данными и взаимодействию с клиентами. Он обрабатывает и исполняет SQL-запросы, отправляет ответы клиентам, управляет транзакциями и обеспечивает целостность информации. Также сервер отвечает за аутентификацию и авторизацию пользователей, контролируя доступ к базе данных.

    SQLite

    SQLite — это быстрая и легкая встраиваемая однофайловая СУБД на языке C, которая не имеет сервера и позволяет хранить всю базу локально на одном устройстве.

    Понятие «встраиваемый» означает, что СУБД не использует парадигму клиент-сервер. Движок SQLite — не отдельно работающий процесс, с которым взаимодействует программа, а библиотека. Программа компонуется с ней, и движок служит составной частью программы.

    Основные особенности SQLite:

    • Легкость: имеет небольшой размер и низкий уровень накладных расходов.
    • Отсутствие клиент-серверной архитектуры: не требует отдельного сервера базы данных. Он работает как библиотека, которая встраивается в приложение.
    • Поддержка SQL: поддерживает большинство стандартных операторов и функций SQL.
    • Транзакции: поддерживает транзакции для обеспечения целостности данных и безопасности операций.
    • Поддержка: может быть использован с различными языками программирования, включая C/C++, Python, Java, JavaScript и многие другие.

    Язык запросов и работа с БД

    Запрос (Query) в контексте баз данных - это команда, отправляемая к базе данных с целью получения определенной информации или выполнения операций с данными. Запросы могут выполнять различные функции, такие как извлечение данных, добавление новых записей, обновление или удаление существующих записей, а также выполнение различных агрегатных операций.

    Создание базы данных и таблиц

    Для создания новой базы данных в языке SQL используется запрос с ключевым словом CREATE с указателем DATABASE:

    CREATE DATABASE имя_базы_данных;

    Создание новой таблицы используется указатель TABLE, после чего в скобке перечисляются столбцы таблицы, их тип данных и особенности поля:

    CREATE TABLE имя_таблицы (
    столбец1 тип_данных [ограничения],
    столбец2 тип_данных [ограничения],
    ...
);

    Чтобы поддерживать максимальную гибкость, строки таблицы, по определению, не должны находиться в каком-то определенном порядке.

    Типы данных в SQLite

    В SQLite каждая колонка в таблице имеет свой тип данных, который определяет тип значений, которые могут быть сохранены в этой колонке:

    • INTEGER: Хранит целочисленные значения. Может быть размером 1-8 байт, в зависимости от значения.
    • REAL: Используется для хранения значений с плавающей точкой.
    • TEXT: Хранит текстовые данные, представленные в виде строки.
    • BLOB: Этот тип данных используется для хранения двоичных данных, таких как изображения, аудио или другие неструктурированные данные.
    • NULL: Этот тип используется для хранения значений NULL.
    • BOOLEAN: SQLite не имеет явного типа BOOLEAN, но обычно значения BOOLEAN представляются как INTEGER, где 0 означает FALSE, а любое ненулевое значение означает TRUE.

    Свойства колонок

    Каждая колонка может обладать рядом свойств:

    • Не пустое (NOT NULL): Это ограничение гарантирует, что в столбце не будет NULL значений.
    • Первичный ключ (Primary Key): Это поле уникально идентифицирует каждую запись в таблице. SQLite автоматически создает индекс для первичного ключа для обеспечения быстрого поиска.
    • Автоинкремент (AUTOINCREMENT): Это позволяет автоматически увеличивать значение первичного ключа при добавлении новых записей. Оно часто используется с первичным ключом INTEGER.
    • Уникальный (Unique): Это ограничение гарантирует, что все значения в столбце будут уникальными.

    Ключи и связи между таблицами

    Первичный ключ (Primary Key)

    Основная функция первичного ключа состоит в том, чтобы обеспечить уникальность записей в таблице и обеспечить быстрый доступ к каждой записи через этот уникальный идентификатор. Без первичного ключа в таблице может быть трудно или невозможно точно определить, на какую запись ссылаться или как ее обновить или удалить.

    Основные свойства первичного ключа:

    • Каждое значение в столбце первичного ключа должно быть уникальным в пределах таблицы.
    • Значения первичного ключа не могут быть NULL.
    • В таблице может быть только один первичный ключ.
    • Индекс обычно кластеризован (сгруппирован) по умолчанию, что означает, что данные в таблице упорядочены в соответствии с первичным ключом, что ускоряет доступ к данным при выполнении запросов.

    Пример создания таблицы с первичным ключом:

    CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    Name VARCHAR(50),
    SurName VARCHAR(50),
    BirthDate DATE,
    Group VARCHAR(10)
);

    Составной ключ (Composite Key)

    Составной ключ является разновидностью первичного ключа и состоит из двух или более столбцов, значения которых в комбинации обеспечивают уникальность записи.

    Основные свойства составного ключа:

    • Может быть несколько таких ключей, и комбинации, из которых они состоят, должны быть уникальными.
    • Состоит из значений нескольких столбцов. Уникальность записи обеспечивается комбинацией значений этих столбцов.
    • Значение каждого столбца в составном ключе не может быть NULL.
    • Также зачастую имеет кластеризованный индекс: данные в таблице располагаются по порядку в соответствии с комбинацией значений столбцов составного ключа.

    Пример создания таблицы с составным первичным ключом:

    CREATE TABLE Enrollments (
    StudentID INT,
    CourseID INT,
    EnrollmentDate DATE,
    PRIMARY KEY (StudentID, CourseID)
);

    Уникальный ключ (Unique Key)

    Так же, как и первичный ключ, связывает уникальное значение с определенными данными. Но есть отличия:

    • Может быть несколько таких ключей, но каждый из них должен быть уникальным.
    • Состоит либо из значения одного столбца, либо из значений нескольких столбцов. Отдельные значения и комбинации должны быть уникальными.
    • Может содержать NULL, но уникальные ключи могут содержать пустое значение.
    • Индекс не кластеризован по умолчанию. Строки могут храниться в таблице в произвольном порядке, а не в порядке значений ключа.

    Пример создания таблицы с уникальным ключом:

    CREATE TABLE Students (
    StudentID INT,
    Name VARCHAR(50),
    SurName VARCHAR(50),
    UNIQUE KEY (StudentID)
);

    Внешний ключ (Foreign Key)

    Внешний ключ связывает записи в одной таблице с записями в другой через общее поле, которое является первичным ключом в другой таблице.

    Основные свойства внешнего ключа:

    • Может быть несколько таких ключей. В одной таблице может быть несколько внешних ключей, связывающих ее с другими таблицами.
    • Может содержать NULL-значения, но только если это разрешено правилами целостности данных.
    • Обычно не кластеризуется. Как правило, не определяет порядок хранения данных в таблице.

    Пример создания таблицы с внешним ключом:

    CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,
    CustomerID INT,
    OrderDate DATE,
    TotalAmount DECIMAL(10, 2),
    FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID)
);

    В этом примере поле CustomerID в таблице Orders является внешним ключом, который ссылается на первичный ключ CustomerID в таблице Customers. Это означает, что каждая запись в таблице Orders будет иметь ссылку на определенного пользователя из таблицы Customers.

    Зачем нужны связи между таблицами?

    Связи между таблицами помогают решить несколько важных задач:

    1. Избежать дублирования данных: Связанные данные будут распределены по разным таблицам без дублирования. Например, вместо хранения адреса клиента в каждой записи заказа можно создать отдельную таблицу «Клиенты» с адресами и связать ее с таблицей «Заказы».

    2. Сделать данные целостными: Информация в базе данных будет корректна и согласована между собой. Например, если клиент отсутствует в таблице «Клиенты», он не сможет сделать заказ.

    3. Оптимизировать запросы: Можно будет создавать сложные запросы для выборки данных, объединяя информацию из нескольких таблиц. Например, если связать таблицы «Клиенты» и «Заказы», можно с помощью одного запроса получить список всех клиентов вместе с заказанными ими товарами.

    4. Повысить производительность: Разделение больших объемов данных на связанные таблицы ускорит поиск и управление информацией. Например, вместо хранения всех данных о клиентах и их заказах в одной огромной таблице данные можно разделить на отдельные связанные таблицы «Клиенты» и «Заказы». Запросы будут выполняться быстрее за счет меньшего объема данных в каждой таблице.

    Правила целостности данных

    В 1970 году математик и информатик Эдгар Ф. Кодд разработал правила целостности для реляционных баз данных. Они обеспечивают корректность и согласованность данных:

    1. Целостность сущностей: Каждая строка в таблице должна быть уникально идентифицирована (обычно с помощью первичного ключа).

    2. Целостность ссылок: Если в таблице есть внешний ключ, то значение этого ключа должно либо соответствовать значению первичного ключа в связанной таблице, либо быть NULL (если это разрешено).

    3. Целостность доменов: Все значения в столбце должны соответствовать определенному домену (типу данных и ограничениям).

    Основные операции SQL

    Добавление данных (INSERT)

    Добавление новой записи в таблицу в SQL происходит с помощью оператора INSERT. После ключевого слова INTO передается название таблицы и полей. После ключевого слова VALUES идут значения, которые должны быть вставлены в поля. Последовательность полей соответствует последовательности передаваемых данных.

    INSERT INTO имя_таблицы (столбец1, столбец2, ...)
VALUES (значение1, значение2, ...);

    Выборка данных (SELECT)

    Запрос SELECT используется для извлечения данных из одной или нескольких таблиц базы данных. Для выбора из какой таблицы делать запрос используется ключевое слово FROM.

    SELECT * FROM имя_таблицы;

    Звездочка ("*") означает выбор всех столбцов.

    Также можно выбирать конкретные столбцы, указав их имена в запросе SELECT:

    SELECT столбец1, столбец2, ... FROM имя_таблицы;

    Вывод данных будет в порядке, в котором были указаны столбцы в запросе.

    Оператор DISTINCT

    Оператор DISTINCT используется для выбора уникальных значений из столбца или комбинации столбцов в таблице. Это означает, что запрос SELECT DISTINCT вернет только уникальные строки в результирующем наборе, и дублирующиеся строки будут удалены.

    SELECT DISTINCT столбец1, столбец2, ... FROM имя_таблицы;

    Условие WHERE

    Запрос с условием WHERE используется для фильтрации строк в результирующем наборе на основе определенного условия. Это условие определяется после ключевого слова WHERE.

    SELECT столбец1, столбец2, ... FROM имя_таблицы WHERE условие;

    В запросе с ключевым словом WHERE могут использоваться реляционные операторы для сравнения (>, <, =, >=, <=, <>).

    Также используются логические операторы: - AND – для получения результата TRUE верными должны быть оба условия - OR – для получения результата TRUE верным должно быть хотя бы одно из условий - NOT – оператор инверсии

    Обновление данных (UPDATE)

    Чтобы изменить существующую запись в таблице базы данных, используется оператор UPDATE. Оператор UPDATE позволяет обновлять значения столбцов в одной или нескольких строках таблицы в соответствии с заданным условием. После ключевого слова SET перечисляются поля и значения, к которым они приравниваются.

    UPDATE имя_таблицы
SET столбец1 = значение1, столбец2 = значение2, ...
WHERE условие;

    Удаление данных (DELETE)

    Чтобы удалить записи из таблицы базы данных, используется оператор DELETE. Оператор DELETE позволяет удалить одну или несколько строк из таблицы в соответствии с определенным условием.

    DELETE FROM имя_таблицы WHERE условие;

    Объединение таблиц (JOIN)

    JOIN в SQL используется для объединения строк из двух или более таблиц на основе определенного условия, которое указывается в предложении ON или USING. JOIN позволяет использовать данные из нескольких таблиц в одном запросе и создавать связи между ними.

    Типы JOIN:

    • INNER JOIN: Возвращает строки, которые имеют соответствующие значения в обеих таблицах. 

      SELECT * FROM таблица1 INNER JOIN таблица2 ON таблица1.столбец = таблица2.столбец;

    • LEFT JOIN: Возвращает все строки из левой таблицы и соответствующие строки из правой таблицы. Если в правой таблице нет соответствующих строк, то будут возвращены NULL значения для столбцов правой таблицы. 

      SELECT * FROM таблица1 LEFT JOIN таблица2 ON таблица1.столбец = таблица2.столбец;

    • RIGHT JOIN: Возвращает все строки из правой таблицы и соответствующие строки из левой таблицы. Если в левой таблице нет соответствующих строк, то будут возвращены NULL значения для столбцов левой таблицы. 

      SELECT * FROM таблица1 RIGHT JOIN таблица2 ON таблица1.столбец = таблица2.столбец;

    • FULL JOIN: Возвращает строки, которые имеют соответствующие значения в любой из таблиц. Если в одной из таблиц нет соответствующих строк, то будут возвращены NULL значения для столбцов этой таблицы. 

      SELECT * FROM таблица1 FULL JOIN таблица2 ON таблица1.столбец = таблица2.столбец;

    Для выбора столбца записывается имя таблицы и через точку передается имя столбца:

    SELECT таблица1.столбец1, таблица2.столбец2 FROM таблица1 JOIN таблица2 ON таблица1.столбец = таблица2.столбец;

    После ключевого слова ON пишется условие соединения вывода.

    DB Browser для SQLite

    Для упрощения работы и наглядной визуализации данных предлагается использование DB Browser.

    DB Browser for SQLite - это бесплатный инструмент с открытым исходным кодом для работы с базами данных SQLite. Это программное обеспечение предоставляет удобный и интуитивно понятный интерфейс для просмотра, редактирования и управления базами данных SQLite.

    Основные функции DB Browser:

    • Создание, открытие и редактирование баз данных SQLite
    • Создание, определение, изменение и удаление таблиц
    • Создание, определение и удаление индексов
    • Просмотр, редактирование, добавление и удаление записей
    • Выполнение SQL-запросов
    • Импорт и экспорт данных в различных форматах
    • Визуализация структуры базы данных

    Подключение библиотеки SQLite

    Для начала работы в проект требуется добавить библиотеку SQLite. Данный пакет можно загрузить из меню управления пакетами NuGet.

    NuGet - это менеджер пакетов для платформы .NET, который позволяет управлять зависимостями и добавлять сторонние библиотеки в проекты на языке C#.

    Для добавления пакета нажмите правой кнопкой по проекту -> Управление пакетами NuGet -> Выберите вкладку обзор и впишите название интересующего пакета. Выбрав пакет, нажмите кнопку установить.

    Для подключения библиотеки SQLite в окно проекта добавьте в поле зависимостей новую using с именем библиотеки:

    using System.Data.SQLite;

    Основные классы для работы с SQLite

    SQLiteConnection

    SQLiteConnection - это класс в пространстве имен System.Data.SQLite, который представляет подключение к базе данных SQLite в C#. Этот класс используется для установления соединения с базой данных SQLite и выполнения операций с ней.

    Для создания новой базы данных используется статический метод CreateFile класса SQLiteConnection. В качестве параметра команды передается полный путь до таблицы:

    SQLiteConnection.CreateFile("C:\\path\\to\\database.db");

    Для подключения к существующей базе данных создается новый экземпляр SQLiteConnection с параметрами, указывающими источник и путь до данной БД:

    string connectionString = "Data Source=C:\\path\\to\\database.db;Version=3;";
SQLiteConnection connection = new SQLiteConnection(connectionString);

    Для открытия и закрытия соединения с базой данных используются методы Open и Close:

    connection.Open();
// Выполнение операций с базой данных
connection.Close();

    SQLiteCommand

    SQLiteCommand - это класс в пространстве имен System.Data.SQLite, который представляет команду SQL для выполнения в базе данных SQLite. Этот класс используется в C# для выполнения SQL-запросов к базе данных SQLite.

    В качестве параметра в конструктор класса передается строка с запросом на языке SQL и объект SQLiteConnection для указания, к какой таблице будет вызван запрос:

    string query = "SELECT * FROM Users";
SQLiteCommand command = new SQLiteCommand(query, connection);

    Прежде чем запрос будет выполнен, должно быть открыто соединение с БД.

    Для создания новой таблицы используется команда CREATE TABLE, которая записывается в строке со всеми параметрами. Далее создается новая команда с данными параметрами и связью с БД:

    string createTableQuery = @"CREATE TABLE Users (
                            UserID INTEGER PRIMARY KEY AUTOINCREMENT,
                            Username TEXT NOT NULL,
                            Email TEXT UNIQUE)";
SQLiteCommand createTableCommand = new SQLiteCommand(createTableQuery, connection);

    Выполнение команды вызывается методом ExecuteNonQuery() – метод выполняет запрос без возвращения данных:

    createTableCommand.ExecuteNonQuery();

    Запросы на INSERT, UPDATE и DELETE также выполняются с помощью ExecuteNonQuery():

    string insertQuery = "INSERT INTO Users (Username, Email) VALUES ('JohnDoe', 'john@example.com')";
SQLiteCommand insertCommand = new SQLiteCommand(insertQuery, connection);
insertCommand.ExecuteNonQuery();

    SQLiteDataReader

    SQLiteDataReader - это класс, который предоставляет возможность последовательного чтения результатов запроса из базы данных SQLite. Он используется для выполнения операций выборки данных из базы данных SQLite.

    Для создания SQLiteDataReader требуется создать запрос и выполнить его методом ExecuteReader():

    string selectQuery = "SELECT * FROM Users";
SQLiteCommand selectCommand = new SQLiteCommand(selectQuery, connection);
SQLiteDataReader reader = selectCommand.ExecuteReader();

    После этого в цикле можно считать данные, получаемые из таблицы, обращаясь по имени SQLiteDataReader и передавая требуемый столбец в квадратных скобках:

    while (reader.Read())
{
    int userId = reader.GetInt32(0); // или reader["UserID"]
    string username = reader.GetString(1); // или reader["Username"]
    string email = reader.GetString(2); // или reader["Email"]

    Console.WriteLine($"UserID: {userId}, Username: {username}, Email: {email}");
}
reader.Close();

    Компонент DataGrid

    В WPF для отображения данных в табличном виде используется элемент управления DataGrid. Он позволяет отображать данные в виде таблицы с возможностью сортировки, фильтрации, редактирования и других операций.

    Привязка DataGrid к данным

    Существует несколько способов связать существующую таблицу и DataGrid.

    Для начала требуется создать класс-структуру для хранения требуемых данных и создать список, состоящий из этой структуры:

    public class User
{
    public int UserID { get; set; }
    public string Username { get; set; }
    public string Email { get; set; }
}

List<User> users = new List<User>();
// Заполнение списка данными
dataGrid.ItemsSource = users;

    Настройка столбцов DataGrid

    Первый способ требует привязки заранее созданных столбцов к данным структуры. Для этого, перейдя во вкладку столбцы в окне свойств компонента, выберите параметр Columns.

    В этом окне требуется создать новый DataGridTextColumn. В окне создания нужно добавить Binding.

    Последний этап – запись параметра, к которому будет привязан столбец. Для этого в окошке XAML в поле колонки требуется прописать параметр в {Binding}. Имя параметра соответствует переменной класса, к которому будет привязана таблица:

    <DataGrid x:Name="dataGrid" AutoGenerateColumns="False">
    <DataGrid.Columns>
        <DataGridTextColumn Header="ID" Binding="{Binding UserID}" />
        <DataGridTextColumn Header="Имя пользователя" Binding="{Binding Username}" />
        <DataGridTextColumn Header="Email" Binding="{Binding Email}" />
    </DataGrid.Columns>
</DataGrid>

    Альтернативно, можно установить свойство AutoGenerateColumns="True", и DataGrid автоматически создаст столбцы на основе свойств привязанных данных.
