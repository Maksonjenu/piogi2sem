Списки

Определение списков

Списки — это динамическая структура данных, которая позволяет хранить элементы разных типов. Списки помогают удобно работать с группами данных, выполняя различные операции — добавление, удаление или изменение элементов.

Для сравнения производительности алгоритмов используется грубое измерение средней производительности и производительности в худшем случае, для обозначения которых используется термин «О» большое.

Значения О

Имя Нотация Описание Пример
Константная O(1) Алгоритм выполняет фиксированное количество операций, независимо от входных данных. Вычисление суммы двух констант.
Логарифмическая O(log N) Алгоритм выполняет количество операций, которое растет логарифмически с ростом входных данных. Часто встречается в алгоритмах поиска в отсортированных массивах. Бинарный поиск элемента в отсортированном массиве.
Линейная O(N) Алгоритм выполняет количество операций, пропорциональное количеству входных данных. Проход по массиву и суммирование всех элементов.
Линейно-логарифмическая O(N log N) Алгоритм выполняет количество операций, которое растет как произведение количества входных данных и логарифма от этого числа. Часто встречается в алгоритмах сортировки. Быстрая сортировка массива.
Полиномиальная O(N ^ 2) Алгоритм выполняет количество операций, которое растет как степень квадрата количества входных данных. Вставка элемента в несортированный массив с использованием перестановки.
Экспоненциальная O(2 ^ N) Алгоритм выполняет количество операций, которое растет экспоненциально с ростом входных данных. Рекурсивное перечисление всех подмножеств набора.
Факториальная O(N!) Алгоритм выполняет количество операций, которое растет как факториал от количества входных данных. Вычисление всех перестановок элементов массива.
  • Вставка в конец - обычно быстрая операция O(1), если не требуется расширение
  • Вставка в середину - требует сдвига всех последующих элементов, O(n)
  • Удаление - также может требовать сдвига элементов для сохранения непрерывности, O(n)

Особенности работы списков

Упорядоченная коллекция — это структура данных, в которой элементы расположены в определённом порядке.

Упорядоченная коллекция означает, что элементы коллекции имеют определённый порядок. Порядок не зависит от значения. Пример - список.

Отсортированная коллекция означает, что в ней не только есть порядок, но и порядок зависит от значения элемента.

Коллекция без какого-либо порядка может сохранять элементы в любом порядке. Пример - множество.

Гомогенность означает, что все значения в наборе имеют одинаковый тип, что обеспечивает структурное единообразие.

Динамическая длина. В зависимости от контекста приложения, списки могут быть заранее определённой длины или изменяемыми в процессе выполнения программы.

Списки (List)

Принцип работы: Упорядоченная коллекция с прямым доступом по индексу.

Особенности:

  • Динамический размер
  • Быстрый доступ к элементам по индексу O(1)
  • Относительно медленные вставка/удаление в середине O(n)
  • Быстрое добавление в конец (амортизированное O(1))

Применение:

  • Хранение последовательностей элементов с частым доступом по позиции
  • Когда важен порядок элементов
  • Для множественных операций просмотра и изменения

Пример в C#:

List<string> names = new List<string>();
names.Add("Алексей");
names.Add("Мария");
string secondName = names[1]; //Доступ по индексу

Стек (Stack)

Принцип работы: LIFO (Last In, First Out) - "последним пришёл, первым ушёл".

Особенности:

  • Доступ только к верхнему элементу
  • Операции Push (добавление) и Pop (извлечение) выполняются за O(1)
  • Нет произвольного доступа к элементам

Применение:

  • Отслеживание вызовов функций
  • Реализация алгоритмов с обратным порядком обработки
  • Проверка корректности скобочных выражений
  • История операций (с возможностью отмены)

Пример в C#:

Stack<int> stack = new Stack<int>();
stack.Push(10);
stack.Push(20);
int lastElement = stack.Pop(); //20

Очередь (Queue)

Принцип работы: FIFO (First In, First Out) - "первым пришёл, первым ушёл".

Особенности:

  • Добавление в конец (Enqueue), извлечение из начала (Dequeue)
  • Операции выполняются за O(1)
  • Нет произвольного доступа к элементам

Применение:

  • Буферизация данных
  • Обработка запросов в порядке поступления
  • Алгоритмы поиска в ширину (BFS)
  • Моделирование реальных очередей

Пример в C#:

Queue<string> queue = new Queue<string>();
queue.Enqueue("Первый клиент");
queue.Enqueue("Второй клиент");
string nextClient = queue.Dequeue(); //"Первый клиент"

Словарь/Хеш-таблица (Dictionary/Hashtable)

Принцип работы: Хранение пар "ключ-значение", где доступ по ключу оптимизирован с помощью хеш-функции.

Особенности:

  • Быстрый поиск, вставка и удаление (в среднем O(1))
  • Ключи должны быть уникальными
  • Отсутствие гарантированного порядка элементов

Применение:

  • Кеширование данных
  • Быстрый поиск по идентификатору
  • Подсчёт частоты элементов
  • Реализация ассоциативных массивов

Пример в C#:

Dictionary<string, int> ages = new Dictionary<string, int>();
ages["Иван"] = 30;
ages["Мария"] = 25;
int mariasAge = ages["Мария"]; //25

Множество (Set/HashSet)

Принцип работы: Коллекция уникальных элементов без определенного порядка.

Особенности:

  • Отсутствие дубликатов
  • Быстрые операции проверки существования O(1)
  • Эффективные операции объединения, пересечения, разности

Применение:

  • Устранение дубликатов
  • Проверка наличия элемента
  • Реализация математических операций над множествами

Пример в C#:

HashSet<int> uniqueNumbers = new HashSet<int>();
uniqueNumbers.Add(5);
uniqueNumbers.Add(10);
uniqueNumbers.Add(5); //Игнорируется, т.к. 5 уже есть
bool contains = uniqueNumbers.Contains(10); //true
