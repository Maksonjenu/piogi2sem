    Диаграммы UML

    UML (Unified Modeling Language): Унифицированный Язык Моделирования

    UML (Unified Modeling Language) — это стандарт для графического представления структуры и поведения объектов в объектно-ориентированных системах. Он позволяет визуализировать сложные системы, делая их более понятными и управляемыми.

    Диаграмма Классов

    Одной из самых востребованных и мощных диаграмм в UML является диаграмма классов. Она служит своеобразным "чертежом" системы, демонстрируя её архитектуру и основные элементы:

    • Классы: Определяют типы объектов в системе, их функциональность и характеристики.
    • Атрибуты: Описывают данные, которыми обладает класс (например, имя, возраст, вес).
    • Методы: Определяют поведение класса, то есть действия, которые он может выполнять (например, вычислитьСумму(), отобразитьДанные()).
    • Взаимосвязи: Отражают отношения между классами (наследование, агрегация, композиция, ассоциация). Эти связи демонстрируют, как классы взаимодействуют друг с другом.

    Структура диаграммы классов

    Диаграмма классов обычно состоит из следующих элементов:

    Классы: прямоугольники, которые представляют собой сущности в системе. Атрибуты: характеристики классов, которые обычно отображаются в средней части прямоугольника. Методы (операции): функции или процедуры, которые определены в классе. Эти элементы располагаются в нижней части прямоугольника. Связи между классами: линии, которые соединяют классы, показывая их взаимосвязь.

    Каждый класс на диаграмме может быть представлен прямоугольником, который разделён на три части:

    1. Первая часть — название класса.
    2. Вторая часть — атрибуты (характеристиски объекта).
    3. Третья часть — методы (поведение объекта).

    Синтаксис элементов диаграммы классов

    Классы

    Каждый класс отображается как прямоугольник, разделённый на три секции. Стандартный вид класса выглядит так:

    Здесь: 

    `+` — публичный элемент (public).
`-` — приватный элемент (private).
`#` — защищённый элемент (protected).

    Атрибуты и методы

    Атрибуты описывают данные, которые могут храниться в классе. Методы — это операции, которые выполняются с данными.

    Пример:

    + name: string
- age: int
# id: string

    name — публичный атрибут типа String. age — приватный атрибут типа int. id — защищённый атрибут типа String.

    Методы отображаются аналогично: 

    + getName(): string - публичный метод, возвращающий строку. Возвращаемое значение string указывается после названия метода.
- setName(name: String): void - приватный метод, устанавливающий значение. Указывается тип данных и название аргумента внутри скобок.
