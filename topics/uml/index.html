    Uml

    UML (Unified Modeling Language): Понимаем? Диаграммы Классов

    UML - стандартный графический язык, который помогает разработчикам "нарисовать" и понять, как устроена программа, особенно если она написана с использованием классов (объектно-ориентированное программирование).

    Зачем нужны диаграммы классов?

    Одна из самых полезных диаграмм в UML — это диаграмма классов. Как бы представляет карту кода:

    • Показывает основные "строительные блоки" программы (классы).
    • Объясняет, из чего состоит каждый блок (его данные и возможности).
    • Демонстрирует, как эти блоки связаны между собой.

    Изучив диаграмму классов можно понять структуру части программы, даже не читая сам код.

    Как читать диаграмму классов: Шаг за Шагом

    Диаграммы классов состоят в основном из прямоугольников (классов) и линий (связей между ними).

    1. Прямоугольник = Класс

    Каждый прямоугольник представляет собой класс — шаблон или "чертеж" для создания объектов в программе. Прямоугольник обычно разделен на три секции:

    • Верхняя секция: Название класса. Это имя "типа" объектов, которые мы описываем.
    • Средняя секция: Атрибуты (данные).
    • Нижняя секция: Методы (действия).

    2. Средняя секция: Атрибуты (Что у класса есть)

    Здесь перечислены атрибуты – характеристики или данные, которыми обладает каждый объект этого класса. Если взять класс "Человек", тогда атрибутами могут быть имя, возраст, цвет глаз.

    Как читать строку атрибута (например, + Name: string):

    • Символ в начале (+, -, #): Это видимость или доступность – кто может получить доступ к этому атрибуту.
      • + (public / публичный): Доступен всем (любой части программы).
      • - (private / приватный): Доступен только внутри самого этого класса. Частый вариант для данных.
      • # (protected / защищённый): Доступен внутри этого класса и его "наследникам" (классам, которые его расширяют).
    • Имя атрибута (Name): Как называется эта характеристика.
    • Двоеточие (:)
    • Тип данных (string): Какого типа информация хранится в атрибуте (текст, число, дата и т.д.).

    Пример чтения атрибутов:

    • + Name: string — Публичный атрибут Name, хранит текст.
    • - age: int — Приватный атрибут age, хранит целое число.
    • # id: string — Защищённый атрибут id, хранит текст.

    3. Нижняя секция: Методы (Что класс умеет делать)

    Здесь описаны методы (или операции) – действия, которые объект этого класса может выполнять. Если класс – "Телефон", то методы могут быть позвонить(), отправитьСмс(), сделатьФото().

    Как читать строку метода (например, + GetTemperature(): double):

    • Символ в начале (+, -, #): Та же видимость, что и у атрибутов – откуда мы можем "вызвать" этот метод.
    • Имя метода (GetTemperature): Название действия.
    • Скобки (): Внутри скобок перечисляются параметры – информация, которая нужна методу для работы. Если скобки пустые, метод не требует дополнительной информации.
      • Пример с параметрами: setName(newName: string) - метод setName требует на вход один параметр newName текстового типа.
    • Двоеточие (:) (если метод что-то возвращает)
    • Возвращаемый тип (double): Какой тип данных метод вернет в результате своей работы. Если метод ничего не возвращает (просто выполняет действие), указывается void.

    Пример чтения методов:

    • + GetTemperature(): double — Публичный метод GetTemperature. Не требует параметров. Возвращает число с дробной частью (double).
    • + TurnLight(): void — Публичный метод TurnLight. Не требует параметров. Ничего не возвращает (void).
    • - calculateInternal(a: int, b: int): int — Приватный метод calculateInternal. Требует 2 параметра - целое число a и b. Возвращает целое число (int).

    4. Линии между прямоугольниками: Связи

    Линии, соединяющие разные классы, показывают, что эти классы как-то связаны или взаимодействуют друг с другом. Например, класс "Заказ" может быть связан с классом "Клиент". Существуют разные типы линий (наследование, ассоциация и т.д.), но для начала достаточно понимать: линия = связь.

    Есть диаграмма:

    Читаем диаграмму "Комната":

    1. Класс: Называется Комната.
    2. Атрибуты:
      • + Name : string — У комнаты есть публичное имя (текст). Любой может его узнать или изменить (если есть соответствующий метод).
      • - temperature : double — У комнаты есть приватная температура (число с дробной частью). Доступ к ней напрямую извне запрещен.
    3. Методы:
      • + TurnLight() : void — Комната умеет публично выполнять действие "Включить свет", которое ничего не возвращает.
      • + GetTemperature() : double — Комната умеет публично сообщать свою температуру (возвращает число).
      • # SetInitialTemperature(temp: double) : void — У комнаты есть защищенный метод для установки начальной температуры; он требует на вход число (temp) и ничего не возвращает. Вероятно, используется только самой комнатой или её наследниками.
